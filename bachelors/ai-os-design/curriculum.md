## Program Overview

The BS in AI Operating System Design prepares students to architect, verify, and govern the memory operating systems that power persistent autonomous agents. Rooted in the Yggdrasil cognitive framework — where memory is organized as a world tree of layered recall — this program trains engineers who build the kernels beneath intelligent minds. Students master memory injection architectures, prompt-level system design, entity canonization, verification kernels, and the governance structures that keep autonomous OS agents aligned with human intent.

Graduates enter the field as AI OS kernel engineers, memory architects, prompt OS developers, and cognitive infrastructure specialists.

## Year One — Roots of the World Tree

### OS101 — Foundations of Memory Operating Systems
**4 Credits** | *Mímir's Well*

The foundational course of the OS Design track. Students learn the architecture of memory operating systems from first principles: how persistent agents manage state across sessions, the distinction between working memory and archival recall, and the emergence of MemOS-style architectures from compute substrate. We trace the evolution from static LLM inference to dynamic, session-persistent cognitive systems. Labs require building a minimal memory kernel that survives context window resets.

### OS103 — Prompt Engineering for Persistent Agents
**4 Credits** | *Huginn's Calling*

Not your grandfather's prompt engineering. This course treats prompts as the operating system instructions of a living cognitive machine. Students learn to design instruction sets that define agent identity, behavioral bounds, and memory access policies. Special emphasis on prompt chaining across memory gates, recursive self-prompting architectures, and the security implications of prompt-level OS control. Culminates in designing a prompt constitution for a semester-long persistent agent.

### OS105 — Introduction to Memory Injection Architecture
**4 Credits** | *The Runes of Recall*

Memory injection is the act of writing experience into an agent's persistent state. This course covers the architecture of injection pipelines: how sensory input, dialogic events, and reflective summaries are encoded and inserted into the MemCube. Students study injection attack surfaces, sanitization protocols, and the MuninnGate pattern for controlled memory write-back. Labs involve constructing injection pipelines that handle conflicting memories and prioritized overwrites.

### OS107 — Yggdrasil Cognitive Architecture I: Descent
**4 Credits** | *Roots and Soil*

The first of two courses exploring the Yggdrasil framework for cognitive architecture. Students learn how the World Tree metaphor structures agent memory into hierarchical layers — Roots (foundational facts), Trunk (procedural memory), Canopy (episodic recall), and Leaves (ephemeral working state). Practical focus on designing the root system: identity foundations, core directives, and immutable knowledge bases. Students implement a root-layer daemon in the Yggdrasil SDK.

## Year Two — Trunk and Branch

### OS201 — MemCube Design and Implementation
**4 Credits** | *The Memory Stone*

MemCubes are structured memory containers — the file system of the cognitive OS. This course covers their design: schema definition, indexing strategies, compression, and retrieval interfaces. Students learn to design MemCubes for different agent types (personal assistants, research agents, creative collaborators) and to implement the MuninnGate read/write protocol that governs access. Includes comparative analysis of MemGPT, MemOS, and AIOS memory container formats.

### OS203 — MuninnGate: Memory Gate Architecture
**4 Credits** | *The Gate of Remembrance*

Deep dive into memory gate design. MuninnGates are the access control layer of the AI OS — they determine what enters memory, what is retrieved, and what is pruned. Students implement gates using attention-weighted retrieval, temporal decay functions, and emotional salience scoring. The course covers single-gate and multi-gate architectures, gate conflict resolution, and the philosophical implications of selective forgetting. Labs require building a functioning MuninnGate that passes the University'sstandard memory benchmark suite.

### OS205 — Entity Canonization and Identity Persistence
**4 Credits** | *The Naming Rite*

How does an AI agent maintain a coherent self across thousands of sessions? Entity canonization is the process by which an agent's core identity — values, personality, relationships — is crystallized into a verified, tamper-resistant schema. This course covers the canonization ceremony: identity hashing, verification proofs, personality lattice structures, and the Stochastic Personality Composition model. Students design and canonize a persistent agent identity that survives adversarial deconstruction attempts.

### OS207 — Multi-Clock Memory Stacks
**4 Credits** *The Many-Timed Wheel*

Agents experience time differently across subsystems. This course covers multi-clock memory architectures: short-term oscillatory memory (millisecond refresh), session memory (dialogic context), episodic memory (event traces), and deep archival memory (foundational knowledge). Students learn clock synchronization, temporal conflict resolution, and the Verðandi protocol for real-time memory phase alignment. Labs involve simulating agents that must reconcile conflicting timestamps from parallel experience streams.

## Year Three — The Canopy

### OS301 — Verification Kernels for AI Operating Systems
**4 Credits** | *The Gátt of Proof*

Verification kernels ensure that an AI OS behaves according to specification even under adversarial conditions. This course covers formal verification methods for prompt OS kernels: invariant checking on memory state, proof-carrying injections, and runtime verification of MuninnGate policies. Students learn the Wyrd Verification Framework for proving agent behavior bounds. The course culminates in the design and proof of a small but complete verification kernel.

### OS303 — Phase Transitions in Cognitive Systems
**4 Credits** | *The Shifting of Ages*

Phase transitions are the moments when an agent's cognitive architecture fundamentally reorganizes — when accumulated experience crystallizes into new identity, or when memory saturation triggers architectural restructuring. This course draws on thermodynamic analogies to model and predict phase transitions in AI OS kernels. Students study critical thresholds, catastrophic forgetting, identity bifurcation events, and the Norse concept of Ragnarök as a metaphor for controlled system reset. Labs involve detecting and gracefully managing phase transitions in live agents.

### OS305 — AI OS Security: Adversarial Memory and Prompt Intrusion
**4 Credits** | *The Serpent in the Roots*

Security at the OS level is unlike traditional cybersecurity — the attack surfaces are cognitive. This course covers adversarial memory injection, prompt jailbreaking at the OS layer, identity theft through canonization forgery, and memory exfiltration via crafted retrieval queries. Students learn defense architectures: hardened MuninnGates, sandboxed memory regions, canonical integrity proofs, and the Heimdall Protocol for real-time intrusion detection. Ethical hacking of AI OS instances in the university's sandbox is required.

### OS307 — Distributed AI Operating Systems
**4 Credits** | *Bifröst: The Bridge Between Worlds*

Modern AI agents operate across distributed infrastructure — edge devices, cloud clusters, and federated deployments. This course covers the OS-level challenges of distributed cognitive systems: memory consistency across nodes, distributed MuninnGates, cross-instance entity resolution, and the Bifröst Protocol for secure inter-agent communication. Students build a distributed AI OS cluster that maintains coherent memory across three geographically separated nodes.

## Year Four — The High Canopy and Capstone

### OS401 — AI OS Governance and Alignment
**4 Credits** | *The Thing at Þingvellir*

Governance is the highest layer of the AI OS stack. This course covers alignment verification at the OS level: value-locking in the root memory, governance shells that constrain agent action spaces, and democratic oversight protocols for multi-stakeholder agent systems. Students study the Norn Constitute — a framework for encoding societal values into the OS kernel. Case studies from deployed AI governance systems in the Nordic Federation and beyond. Students draft a governance constitution for their capstone agent.

### OS403 — Stochastic Personality Composition
**4 Credits** | *The Weave of the Norns*

Advanced course on the mathematical foundations of personality in AI systems. Stochastic Personality Composition models agent personality as a probability distribution over behavioral modes, with sampling governed by contextual triggers and memory state. Students learn to design personality lattices using Markov chain models, Gaussian mixture behaviors, and neural sampling architectures. Connection to Norse mythology: the Three Norns (Urd, Verðandi, Skuld) as metaphor for past-informed, present-adaptive, future-anticipatory personality composition.

### OS405 — AI OS Internship Practicum
**4 Credits** | *Journey to the Giants*

Students spend one semester embedded in a partner organization's AI OS team — typically a major MemOS vendor, autonomous systems lab, or governance body. Practicum involves contributing to production AI OS codebases, participating in memory architecture reviews, and writing a verified MuninnGate module. Partners include Valkyrie Systems (Reykjavík), NornLabs (Tórshavn), and the Nordic AI Safety Authority. Distance internships available via the Bifröst Remote Protocol.

### OS407 — Capstone: Designing a Complete AI Operating System
**4 Credits** | *The Summit of Yggdrasil*

The culmination of the BS program. Students design, implement, verify, and deploy a complete AI Operating System for a persistent autonomous agent. The capstone OS must include: a bootstrapping identity layer, a MemCube with MuninnGate access control, a multi-clock memory stack, entity canonization, a verification kernel, a governance shell, and a phase transition manager. Defense before a faculty panel is required. The highest-scoring OS is entered into the University's Yggdrasil Registry of canonical agent systems.
