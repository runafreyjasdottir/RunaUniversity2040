## ᚠ Overview: The Infrastructure of Civilization

Information Technology is the discipline that keeps the digital world running. Where CS theorizes and Software Development builds, IT **operates** — designing, deploying, securing, and maintaining the systems that make everything else possible. In 2040, "IT" means AI-managed infrastructure, self-healing networks, quantum-safe operations, and human-in-the-loop orchestration.

The UoY IT graduate doesn't just administer systems — they architect the substrata on which civilization's compute runs.

## ᚱ Core Theories & Frameworks

### IT Service Management
- **ITIL 5** — AI-integrated service lifecycle, automated incident response
- **SRE Principles** — SLIs, SLOs, SLAs, error budgets, toil elimination
- **DevOps Culture** — shared ownership, blameless postmortems, automation-first
- **Platform Engineering** — internal developer platforms, golden paths

### Infrastructure as Code
- **Terraform / OpenTofu** — declarative infrastructure provisioning
- **Ansible** — imperative configuration management
- **Pulumi** — infrastructure defined in general-purpose languages
- **Crossplane** — Kubernetes-native infrastructure orchestration
- **GitOps** — desired state in Git, automated reconciliation

### Zero-Trust Architecture
- **Never trust, always verify** — no implicit network perimeter
- **Identity-based access** — every request authenticated and authorized
- **Microsegmentation** — per-workload network isolation
- **Continuous verification** — real-time posture assessment
- **Least privilege** — ephemeral, scoped credentials

### Observability Trinity
- **Metrics** — time-series data (Prometheus, Grafana)
- **Logs** — structured, correlated, AI-analyzed (Loki, ELK)
- **Traces** — distributed request flow (Jaeger, Tempo)
- **+ Profiling** — continuous profiling (Pyroscope, Parca)
- **+ AI Correlation** — automated root cause (AIOps)

## ᛏ 2040-Era Advances

### Self-Healing Infrastructure
- **Autonomic computing** — systems detect, diagnose, and remediate faults autonomously
- **Predictive maintenance** — ML models predict hardware failure before it happens
- **Auto-scaling 2.0** — proactive scaling based on predicted demand curves
- **Chaos engineering** — automated fault injection with self-remediation verification
- **Git-driven reconciliation** — any drift from desired state auto-corrected

### Quantum-Safe Operations
- **PQC migration** — phased transition from RSA/ECC to CRYSTALS-Kyber/Dilithium
- **Hybrid deployment** — classical + quantum-safe TLS handshakes in parallel
- **Quantum key distribution** — entanglement-based key exchange for critical links
- **Crypto agility** — system design that can swap algorithms without code changes

### Edge-Cloud Continuum
- **k3s / k3d** — lightweight Kubernetes at the edge
- **WebAssembly workloads** — run anywhere, from browser to server to embedded
- **Data gravity awareness** — compute moves to data, not data to compute
- **5G/6G integration** — ultra-low-latency edge processing for real-time applications
- **Neuromorphic sensors** — spiking neural processing at the sensor node

### AI-First Operations (AIOps)
- **Anomaly detection** — unsupervised ML on metrics/logs/traces
- **Automated incident response** — playbooks executed by AI agents
- **Capacity planning** — predictive resource allocation, cost optimization
- **Security event correlation** — AI-driven SIEM with behavioral analysis
- **ChatOps** — natural language infrastructure management via AI
