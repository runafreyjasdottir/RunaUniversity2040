# Bachelor of Science in Software Development — University of Yggdrasil, 2040

## ᚠ Overview: Code as Craft

Software Development at UoY is the art and science of building reliable, maintainable, and ethical software systems. Where Computer Science peers into the theoretical foundations, Software Development focuses on **the making** — the disciplined craft of turning specifications into working, tested, deployed code that humans can understand and machines can execute safely.

By 2040, software development is AI-augmented at every stage, but the developer remains the architect. AI writes first drafts; the developer designs, verifies, and owns the final system.

## ᚱ Core Theories & Frameworks

### Software Design Principles
- **SOLID** — updated for 2040 with AI-generated code considerations
- **Domain-Driven Design (DDD)** — bounded contexts now map to microservice boundaries and agent domains
- **CQRS + Event Sourcing** — write/read separation with immutable audit logs
- **12-Factor App** — extended for edge-cloud continuum (15-Factor in 2040)
- **Clean Architecture** — dependency inversion, the dependency rule, framework independence

### Development Methodologies (2040)
- **AI-Augmented Agile** — LLM generates sprint backlog, humans prioritize and verify
- **Trunk-Based Development** — short-lived branches, automated merge, feature flags
- **Shift-Left Security** — security from commit 0, not after penetration
- **Contract-First Development** — API specs (OpenAPI/gRPC) before implementation
- **Self-Documenting Code** — structured comments → auto-generated docs, never stale

### Testing Philosophy
- **Test Pyramid** — unit → integration → E2E, AI generates unit tests
- **Property-Based Testing** — QuickCheck/Hypothesis for exhaustive edge-case discovery
- **Mutation Testing** — verify test quality by injecting faults
- **Snapshot/Golden Testing** — deterministic regression for UI and API
- **Chaos Engineering** — Gremlin/Litmus for production resilience
- **Formal Verification** — Lean 4 for safety-critical modules

## ᛏ 2040-Era Advances

### AI-Pair Programming
- **Local coding assistants** — 7B-30B parameter models run on developer laptops
- **Spec-to-code pipelines** — natural language → verified implementation
- **Automated PR review** — AI checks for logic errors, security issues, style
- **Self-healing CI** — build failures auto-diagnosed and patched by agents
- **Semantic code search** — find code by meaning, not just text

### Post-Container Deployment
- **MicroVMs** — Firecracker/kata for fast, isolated, lightweight VMs
- **WASM everywhere** — edge, serverless, browser, embedded, all from one compilation
- **Service mesh 2.0** — zero-knowledge proof-based service identity
- **GitOps + progressive delivery** — ArgoCD, Flagger, canary-by-metrics
- **Edge-cloud continuum** — deploy once, schedule anywhere (k3s at edge, k8s in cloud)

### Verified & Safe Software
- **Proof-carrying code** — compiled artifacts ship with machine-checked proofs
- **Capability-based security** — every module declares minimum privileges
- **Memory-safe defaults** — Rust/Roofed adoption >60% of new systems code
- **Supply chain verification** — Sigstore, SBOMs, reproducible builds mandatory
- **AI code auditing** — agents continuously scan repositories for vulnerabilities
