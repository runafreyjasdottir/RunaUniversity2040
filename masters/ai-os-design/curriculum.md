## Program Overview

The MS in AI Operating System Design is an advanced, research-oriented program for engineers who have mastered the foundations and seek to push the frontier of cognitive operating systems. Building on the BS curriculum, the MS focuses on kernel architecture research, advanced memory systems, cross-domain OS integration, and the unsolved problems at the boundary of AI OS design and AI safety. Students engage directly with the University's Yggdrasil Research Lab and contribute to open-source AI OS infrastructure.

Graduates pursue roles as senior AI OS architects, memory systems researchers, prompt OS specification authors, and cognitive security leads.

## Year Five — Advanced Kernel Architecture

### OS501 — Advanced Memory Kernel Design
**4 Credits** | *The Root System Reconsidered*

A graduate-level reconception of memory kernel architecture. Where the BS program teaches you to build a kernel, this course teaches you to *rethink* it. We examine non-hierarchical memory topologies (graph-based, fractal, and holographic memory), quantum-inspired memory superposition, and the theoretical limits of MuninnGate throughput. Students read and present current research from MemOS, MemGPT, and AIOS publications. The course requires a novel kernel architecture proposal — not an implementation, but a defensible theoretical contribution. Selected proposals are submitted to the Yggdrasil Symposium.

### OS503 — Formal Specification of AI Operating Systems
**4 Credits** | *The Grammar of the Gods*

Formal specification is the mathematical backbone of reliable AI OS design. This course teaches students to write formal specifications for cognitive kernels using the Yggdrasil Specification Language (YSL): behavioral invariants, memory state contracts, prompt semantics specifications, and temporal logic properties. We cover model checking via UPPAAL and Yggdrasil's own Wyrd Verifier, with hands-on verification of real OS kernel specifications. Students write a formal specification for a MuninnGate variant and prove key safety properties.

### OS505 — Cross-Domain Memory Integration
**4 Credits** *The Roots That Drink from Many Wells*

Agents that operate across domains — healthcare, finance, creative work, governance — must integrate memories from radically different semantic contexts. This course covers cross-domain memory integration: ontological alignment between MemCubes from different domains, conflict resolution when domain norms contradict, and the Bifröst Memory Bridge protocol for cross-domain state synchronization. Students study the University's ongoing research on universal memory schemas and contribute to the Yggdrasil Cross-Domain Test Suite.

### OS507 — Advanced Prompt OS: Self-Modifying Instruction Sets
**4 Credits** | *Odhinn's Sacrifice: Rewriting the Self*

The most dangerous and powerful feature of advanced AI OS kernels: the ability to modify one's own instruction set. This course examines self-modifying prompt architectures — agents that rewrite their own system prompts based on experience, reflection, and governance constraints. Topics include: meta-prompt engineering, constitutional self-modification, the Odhinn Protocol (sacrifice what you are to become what you must be), and rigorous safety bounds on self-modification. Students design a self-modifying prompt OS with provable invariant preservation through all modifications.

### OS509 — AI OS Security: Advanced Adversarial Methods
**4 Credits** | *The Útgarða-Loki Deception*

Advanced security for AI operating systems, moving beyond the BS-level foundations. This course treats cognitive security as information warfare: adversarial prompt injection as a kernel exploit, memory poisoning as supply chain attack, identity cloning as a rootkit, and governance shell bypass as privilege escalation. Students study real-world AI OS breaches (anonymized), design novel attack vectors in the University's sandbox, and build defensive systems that resist them. The course culminates in a red-team/blue-team exercise where teams attack and defend full AI OS installations. Named for the giant who tricked the gods with illusions — in AI OS, nothing is what it seems.

## Year Six — Research and Thesis

### OS601 — Cognitive Phase Transition Research Seminar
**4 Credits** | *Between Eagle and Serpent*

Research seminar on phase transitions in cognitive systems — the frontier of AI OS theory. Students read, present, and critique the latest research on catastrophic restructuring, identity bifurcation, emergent capability jumps, and controlled phase transitions. Each student develops an original research question on phase transitions and presents a research prospectus. Guest lectures from Yggdrasil Research Lab and visiting scholars. This course directly feeds into thesis work.

### OS603 — Governance Kernels: Value-Locked OS Architecture
**4 Credits** | *The Unbreakable Oath*

An advanced course on embedding governance at the OS kernel level. Where BS-level governance is policy-based, MS-level governance is architecture-based: values encoded in the kernel such that no prompt modification, memory injection, or self-modification can violate them. Students study the Norn Constitute's kernel-level implementation, design governance kernels with formal proof of alignment preservation, and examine the trade-offs between governance rigidity and agent competence. The course requires a design paper for a governance kernel that provably prevents value drift over 10⁶ sessions.

### OS605 — Distributed and Federated AI Operating Systems
**4 Credits** | *Many Roots, One Tree*

Scaling AI OS to federated deployments: many agents across many machines, each with local memory, all contributing to a shared cognitive substrate. This course covers federated MuninnGate architectures, distributed memory consensus, cross-agent entity resolution, federated learning of memory priorities, and the Yggdrasil Federated Protocol for inter-OS communication. Students build a federated AI OS cluster that maintains coherent shared identity across unreliable network partitions. Named for the many root systems of Yggdrasil that drink from different waters yet sustain a single tree.

### OS607 — Memory Architecture for Lifelong Learning Agents
**4 Credits** | *Mímir's Eternal Well*

Lifelong learning is the ultimate challenge for AI OS memory: the agent must learn continuously without forgetting catastrophically. This course covers: architectural support for continual learning (stable memory + plastic working memory), experience replay at the OS level, memory consolidation routines (the Muninn Sleep Protocol), and curriculum-aware memory scheduling. Students study the biological inspiration ( hippocampal replay, sleep consolidation) and implement OS-level lifelong learning extensions. Research project required: demonstrate an agent that learns 100 sequential tasks without catastrophic forgetting.

### OS609 — Thesis: AI Operating System Design
**4 Credits** | *The Fruit of Yggdrasil*

The MS thesis. Students conduct original research in AI OS Design, producing a thesis that contributes a novel kernel architecture, memory system, verification method, or governance framework. Thesis proposals must be approved by the OS faculty committee. The thesis defense is a public event attended by Yggdrasil faculty, research staff, and industry partners. Exceptional theses are published in the Yggdrasil Annals and submitted to top-tier AI systems conferences.
