## ᚱ Research Areas

### Developer Experience & Productivity
- AI-augmented coding: copilots, code generation, automated debugging
- Program comprehension: how developers understand code
- Cognitive load metrics: measuring mental effort during programming
- Editor/IDE design: reducing friction, increasing flow state
- Onboarding & learning: how quickly new developers become productive

### Programming Language Design
- Type system innovation: algebraic effects, dependent types, linear types
- DSLs for specific domains (finance, games, quantum, hardware)
- Error messages as first-class UX: human-readable diagnostics
- Gradual typing: smooth migration between typed and untyped
- Memory safety without garbage collection: Rust, Vale, Hylo

### Software Verification
- Verified compilers: machine-checked correctness of code generation
- Contract-based programming: pre/post-conditions, invariants
- Model checking: exhaustive verification of concurrent systems
- Proof assistants for software: Coq, Lean 4, Dafny
- Runtime verification: monitoring safety properties at execution time

### Empirical Software Engineering
- Mining software repositories: what bugs, features, and patterns emerge?
- Controlled experiments: does technique X actually improve outcomes?
- Reproducibility: can others rebuild and verify your results?
- Human studies: developer behavior under controlled conditions
- AI code evaluation: how good is AI-generated code, really?

### AI-Native Development
- Specification-to-implementation: natural language → verified code
- Self-healing code: AI detects and patches runtime errors
- Architecture synthesis: AI proposes software architectures from requirements
- Testing generation: AI creates comprehensive test suites
- Documentation generation: always up-to-date, always correct
